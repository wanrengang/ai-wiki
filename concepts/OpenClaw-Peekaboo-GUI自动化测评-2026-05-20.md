# 看屏幕、用键鼠，OpenClaw「睁眼」了 —— Peekaboo GUI 自动化深度测评

> 来源：AI科技评论（2026-05-20）
> 作者：宇景
> 原始链接：https://mp.weixin.qq.com/s/2PZC5PV5Zv-w1clNFcyptw

## 概述

2026 年 5 月 11 日，本地 AI 智能体框架 OpenClaw 正式上线 macOS 专属桌面操控工具 **Peekaboo**，提供像素级截图、UI 元素识别、以及完整的 GUI 自动化能力。

简单来说，就是让 Agent "长出眼睛"——看得见屏幕、点得到按钮，自主操作电脑。

---

## Peekaboo 核心能力

### 1. 像素级识别
- 自动精准标注屏幕上的按钮、输入框、弹窗、软件窗口等所有界面元素
- 无需人工标记或提前适配

### 2. 完整键鼠模拟
- 自主完成鼠标点击、文字输入、页面滚动、快捷键操控等全套键鼠行为

### 3. 无依赖通用性
- 不依赖软件官方 API
- 无需嵌入 SDK 适配
- 所有桌面软件、网页平台都能直接兼容
- 在赛博世界留下的痕迹就像真正的人类用户

---

## 实测案例

### 案例一：信息检索
要求：打开 Safari → 自动关闭网页弹窗干扰 → 顶部搜索框查找「AI近期大事」→ 打开第一个网站 → 关闭网站

覆盖能力：图标识别、按钮点击、弹窗检测、搜索框定位、内容输入

**结果**：整套流程全部由 OpenClaw 自主完成，规避弹窗干扰、适配动态网页布局。

### 案例二：跨软件长链路复合任务
要求：Safari 搜索百度热搜榜 → 依次提取热搜榜、电影榜、小说榜、电视剧榜前六名 → 回到桌面新建 Excel 表格汇总数据

**结果**：涉及浏览器、办公软件、系统桌面、文件夹、系统设置等多个操作平台之间的跳转，全程无人工干预卡顿。

> OpenClaw 的定位正在从 AI 工具的调用者，转向下一代桌面环境操作底座。

## 当前局限性

### 1. 坐标偏移
不同的显示器分辨率、窗口缩放比例之下，页面元素位置变化偶尔会导致点击坐标漂移。

### 2. 上下文遗忘
超长程任务进行到后半段时，常常忘记自己此前干了什么，重复执行某些操作直到卡死进程。

### 3. 执行速度慢
复杂界面识别、超长任务流程导致比较明显的延迟，这是 GUI 自动化工具的通病。

### 4. Token 成本
视觉方案在执行速度上无法与代码命令直接操作比拟。Peekaboo 通过 VQA 定向分析和元素 ID 压缩减少 Token 消耗，但仍是一笔无法忽视的成本。

## 战略意义：必经之路还是权宜之计？

### 视觉方案的优势
- 突破封闭生态：不依赖厂商 API，即使多年不更新的小众软件和从未对外开放接口的内部工具都被一视同仁纳入能力范围
- 多模态技术迭代红利：UI 识别曾是长期困扰 GUI 自动化的瓶颈，多模态技术突破让 Peekaboo 成为第一款普通人也能用的桌面自动化工具

### 豆包的教训
豆包手机助手曾因使用安卓系统级高危权限模拟用户点击、跨应用跳转，被微信、支付宝、淘宝等主流 App 的风控系统识别为异常行为，导致部分跨应用操作功能受限或被迫下线。

### 核心问题
视觉方案到底是打通 AI 现实操作链路的必经之路，还是生态封禁被真正突破之前的权宜之计？速度、成本、准确性，日后或许都会成为 Peekaboo 的命门。

---

## 总结

Peekaboo 让 OpenClaw 成为了距离接管整个电脑最近的选手。但坐标偏移、上下文遗忘、执行速度、Token 成本等问题仍需解决。

当 Agent 之争走向落地执行能力的较量，视觉方案与代码命令方案的路线之争才刚刚开始。
