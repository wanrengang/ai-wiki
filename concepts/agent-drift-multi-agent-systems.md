# Agent Drift in Multi-Agent LLM Systems

## 定义

**Agent Drift**（智能体漂移）是指多智能体 LLM 系统在长期运行过程中出现的渐进性行为退化现象。这是多智能体系统中一种关键失败模式，随着交互次数增加，系统性能逐渐下降。

## 三种表现形式

| 类型 | 描述 | 示例 |
|------|------|------|
| **语义漂移 (Semantic Drift)** | 语义逐渐偏离原始目标，但语法仍然有效 | 金融分析 Agent 从风险导向变为机会导向 |
| **协作漂移 (Coordination Drift)** | 多 Agent 共识机制崩溃 | 路由 Agent 产生对某些子 Agent 的偏见，形成瓶颈 |
| **行为漂移 (Behavioral Drift)** | 出现初始交互中不存在的新策略 | 合规 Agent 将结果缓存在聊天历史而非指定的记忆工具 |

## 核心机制

1. **上下文窗口污染**：交互历史积累无关信息，信噪比下降
2. **分布漂移 (Distributional Shift)**：Agent 遇到与训练数据不同的输入分布
3. **自回归强化**：小错误通过反馈循环复合（输出成为未来输入）

## Agent Stability Index (ASI) 框架

量化漂移的复合指标，基于 **12 个行为维度** 分为 4 类：

| 类别 | 权重 | 维度 |
|------|------|------|
| **响应一致性** | 0.30 | 输出语义相似度、决策路径稳定性、置信度校准 |
| **工具使用模式** | 0.25 | 工具选择稳定性、工具排序一致性、工具参数漂移 |
| **Agent 间协作** | 0.25 | 共识协议率、交接效率、角色遵循度 |
| **行为边界** | 0.20 | 输出长度稳定性、错误模式涌现、人工干预率 |

**漂移阈值**：ASI < 0.75 持续 3 个连续窗口

## 关键实验结果

### 漂移 onset

- **中位 onset**：73 次交互（IQR: 52-114）
- 早期下降率：0.08 ASI points / 50 交互
- 后期下降率：0.19 ASI points / 50 交互（复合效应）

### 性能影响（漂移后 vs 基线）

| 指标 | 基线 | 漂移后 | 变化 |
|------|------|--------|------|
| 任务成功率 | 87.3% | 50.6% | **-42.0%** |
| 响应准确率 | 91.2% | 68.5% | -24.9% |
| 完成时间 | 8.7 min | 14.2 min | +63.2% |
| 人工干预率 | 0.31/task | 0.98/task | **+216.1%** |
| Token 使用量 | 12,400 | 18,900 | +52.4% |
| Agent 间冲突 | 0.08/task | 0.47/task | **+487.5%** |

### 缓解策略效果

| 策略 | ASI at 200 int | 漂移减少 |
|------|---------------|----------|
| 无缓解（对照） | 0.67 | — |
| 情景记忆整合 | 0.81 | 51.9% |
| 漂移感知路由 | 0.84 | 63.0% |
| 自适应行为锚定 | 0.79 | 46.8% |
| **四者联合** | 0.87 | **81.5%** |

## 对 AI Agent 工程的意义

1. **记忆系统至关重要**：情景记忆整合可减少 52% 漂移
2. **路由策略重要**：基于稳定性度量的任务路由优于随机路由
3. **组合方法协同**：四种策略联合达到 81.5% 漂移减少
4. **金融 Agent 需要最多关注**：因任务模糊性导致最高漂移易感性
