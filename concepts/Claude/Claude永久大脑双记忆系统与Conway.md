## 一、Memory Files（文件记忆）—— 内置个人 Wiki

### 现有问题：经典记忆 = 滚动便签纸
信息量一大就开始"溢出"——旧的被新的覆盖，重要的被琐碎的淹没。记住了昨天点的咖啡口味，却忘了上个月花三小时讨论的产品架构。

### Memory Files 的突破
Anthropic 给 Claude 造了一个内置「个人Wiki」，彻底颠覆：

| 优势 | 说明 |
|------|------|
| 容量无限 | 基于文件系统，理论上可无限扩展，不再受单一摘要长度瓶颈 |
| 精准检索 | 聊代码时只调取技术文档，聊旅行时只调取出 行偏好——按需检索 |
| 用户控制 | 可像编辑 Wiki 一样随时浏览、修改、删除任何记忆文件 |

**本质**：Anthropic 把 OpenClaw/Hermes 等长时运行 AI 智能体已有的架构，做成了消费级产品的原生功能。

## 二、Dreams（做梦机制）—— 记忆的 REM 睡眠

灵感来自人类神经科学中的 REM 睡眠阶段——大脑在休息时回放白天经历、强化重要记忆、丢弃无用噪声、整合短期为长期记忆。

### 功能
Claude 在两次工作会话之间「闲下来」时，Dreams 自动启动，对记忆文件深度整合：
- **合并重复项**：把分散在不同文件的同一话题信息归拢
- **替换过时条目**：「昨天决定用Redis」→「2026年5月15日决定用Redis」
- **解决逻辑矛盾**：保留更新、更可靠的记忆
- **挖掘隐藏模式**：发现实时对话中都没注意到的规律

### 触发条件
- 至少累积 5 次对话
- 或距离上次整合超过 24 小时
- 或手动输入 `/dream`

### 实战效果（Netflix、Rakumen、Wisedocs 等首批企业）
- 首次处理错误率暴降 **97%**
- 文档验证提速 **30%**

## 三、Conway——7×24h 永不下线的 Agent 平台

### 是什么
Anthropic 酝酿中的下一代杀手级产品，一个永不下线的 AI 智能体平台。在 Claude 界面侧边栏以独立入口出现。

### 核心功能
- **独立运行环境**，设计为常驻后台运行
- 监听外部事件、主动触发任务
- 通过 Webhook 接收信号
- 操控浏览器、运行 Claude Code
- 支持 CNW ZIP 自定义扩展包格式

### 三个核心功能区
- 搜索（Search）
- 对话（Chat）
- 系统（System）

### 安全优势 vs OpenClaw
OpenClaw 头两个月就被发现 **9 个 CVE**，**4.2 万**暴露实例在公网裸奔。

Conway 跑在 Anthropic 托管云基础设施上：
- 扩展必须显式安装
- Webhook 可逐服务开关
- 浏览器集成走 Claude 权限模型

> "OpenClaw 直接对标，但 Conway 是 Anthropic 原生的，直接跑在 Claude 上面，安全性和整合度完全不在一个量级。"

## 四、为什么此时推出 Memory Files？

**为 Conway 铺路。**

永不下线的 AI 智能体最需要的是：不是一条随时被覆盖的滚动摘要，而是一套可以无限扩展、按需检索、自动维护的持久记忆系统。

- Memory Files → 提供存储架构
- Dreams → 提供维护机制
- Conway → 提供永不下班的运行时

三者叠加 = 从记忆到反思，再到行动的完整闭环。
