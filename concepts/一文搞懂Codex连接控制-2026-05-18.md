# 一文搞懂Codex连接控制：iPhone，MacBook，Mac mini，远程服务器

> 来源：AI寒武纪
> 日期：2026年5月18日 18:43

## 核心场景

Codex "连接"主要分三类，用好以后可以手拿iPhone随时随地干活，和女朋友逛街等她时也可以 vibe coding，相当于用 Codex 连接起整个开发网络，共享上下文，好用到飞起。

## 三种连接类型

### 1. Control this Mac

**定义**：把这台 Mac 当作 Codex 主机，让手机或其他设备来控制它。

**典型场景**：你人在外面，用 ChatGPT iOS/安卓 里的 Codex 继续 Mac 上的项目、查看终端输出、审批命令、看 diff、继续线程。

**使用方法**：
1. 在 Mac 的 Codex App 里开启"允许其他设备连接"
2. 用手机 ChatGPT 扫 Codex App 提供的二维码完成绑定
3. 保持 Mac 在线、醒着、Codex App 正在运行
4. 手机里进入 Codex，就能选择这台 Mac 继续工作

---

### 2. Control other devices

**定义**：用当前这台 Mac 去控制另一台 Codex 主机（反过来）。

**典型场景**：
- 家里 Mac mini 常开，项目和环境都在那里
- 你现在用笔记本打开 Codex，连接那台 Mac mini
- 之后可以在笔记本上继续 Mac mini 上的线程、项目、命令和工具

---

### 3. SSH

**定义**：让 Codex 连接远程开发机/服务器，在远程文件系统和 shell 上工作。

**与前两者的区别**：前两个是 Codex App 主机之间的远程控制；SSH 是把项目跑到远程机器上，比如 devbox、云服务器、公司开发机。

**使用方法**：
1. 先在本机 `~/.ssh/config` 配好明确的 Host
2. 确认本机能直接连：`ssh devbox`
3. 在远程主机上安装并登录 Codex，确保远程 shell 里能找到 `codex` 命令
4. 回到 Codex App：Settings > Connections > SSH，添加或启用这个 host，然后选择远程项目目录

---

## 一句话总结

- `Control this Mac`：手机/别的设备遥控这台 Mac
- `Control other devices`：这台 Mac 遥控别的 Codex 设备
- `SSH`：Codex 进入远程服务器/开发机干活

**注意**：Codex 同步的是 Codex 主机上的项目和线程上下文
