# 钉钉悟空

## 概述

阿里巴巴企业级 AI 原生工作平台，由钉钉团队打造。战略定位为"C端千问，B端悟空"。

## 关键事实

| 项目 | 内容 |
|------|------|
| 定位 | 企业级 AI 原生工作平台 |
| 战略 | C端千问，B端悟空 |
| CLI 开源 | 2026-03-27，Apache-2.0 协议 |
| 核心能力 | 1000+ 核心能力 CLI 化 |

## 核心能力

### 五大真实场景

| 场景 | 能力 |
|------|------|
| 全网搜索→文档→分享 | 一条指令从搜索到产出到分享，全链路闭环 |
| 读知识库写方案 | 跨多文档交叉理解，基于企业已有沉淀生成新方案 |
| 会议结束全搞定 | 整理纪要→拆待办→创建任务→@相关人，30秒闭环 |
| 竞品看板写入表格 | 搜索→筛选→结构化→写入钉钉表格→发群通知 |
| 一人团队自动化 | 选品/上架/营销全链路，1688+小红书+淘宝数据联动 |

## 与龙虾（OpenClaw）对比

| 问题 | 龙虾 | 悟空 |
|------|------|------|
| AI 能碰到企业数据吗？ | 只能操作本地文件 | 直接操作钉钉全量能力 |
| 产出归谁？能管吗？ | 文件散落本地 | RealDoc 系统，产出归企业、可追溯 |
| 企业本身用钉钉吗？ | 通用工具 | 天然打通，开箱即用 |

## 核心洞察

> "如果说龙虾是脱缰的野马，悟空就是戴着紧箍咒的行者。"

- 紧箍咒不是限制，是企业敢让 AI 真正碰数据的前提
- AI 产出的每份文件自动归属企业，完整归属记录
- 阿里全系能力正以 Skill 形式接入

## 金句

> "过去是人用钉钉来工作，未来是 AI 用钉钉来工作。"

> "安全、合规、可管理，不是加分项，是入场券。"
