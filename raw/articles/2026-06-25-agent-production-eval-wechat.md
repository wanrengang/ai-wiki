# Agent 从「优等生」到「好员工」还差什么？

**来源：** 机器之心 PRO会员通讯
**发布日期：** 2026-05-24
**摘要：** Agent 进入企业生产环境后评估体系分析

## 核心洞察

### Agent benchmark 的定位变化
- 过去一年，Agent 在公开演示和 benchmark 中不再只是回答问题
- 网页浏览、代码修改和软件环境操作等连续任务成为主要能力衡量方式
- 业界能力指标从单次答案质量，扩展到任务完成率、工具调用、执行过程和状态维护等维度

### 高分与可部署之间的落差
- 企业容易将高分 Agent 与可部署生产力联系起来，但实际体验并不总是跟着跑分走
- 进入真实账号、内部数据、业务流程和人工审查链路后，仍可能出现错误、低效或难以追责的问题
- Galileo 和 Datadog 报告：问题指向生产环境中的 eval、可靠性和链路观测

### Benchmark 的局限性（Springer 15个主流Agent benchmark综述）
- 没有 benchmark 将安全性或安全防护纳入评分
- 没有 benchmark 将成本效率纳入主要评估协议
- 15个 benchmark 中有13个主要依赖二元成功指标，能判断任务是否完成，但较少说明执行过程是否稳定、可控、可复现

### 生产环境三大风险缺口
1. **数据误写** — Agent 接触账号权限、业务数据后的误操作
2. **链路中断** — 执行链路缺乏记录导致故障难以复现
3. **合规缺口** — AI 进入工程组织后，审查、修复和切换工具的时间成本

### 生产评估三层体系
1. **上线前行为测试** — 测试覆盖率、测试规格、发布门禁
2. **运行中链路观测** — 运行轨迹、工具调用、模型路由、延迟、token消耗、成本和服务容量
3. **事故后组织指标** — 审查、修复、切换工具、开发者信任

### Harness Engineering 的角色
- 让系统更快暴露错误、定位错误并推动修正
- 解决 Agent 如何更稳定地运行，不等于完整的生产评估
- 执行框架仍需生产评估体系配合
