# OpenClaw 多飞书机器人记忆体系设计

> 如何设计多个飞书机器人的记忆体系：共享、独立或混合。

## 三种记忆模式

### 模式 1：统一记忆（所有机器人共享）

**架构：**
```
所有飞书机器人 → 共享同一个记忆库 → ~/.openclaw/workspace/
```

**特点：**
- ✅ 所有机器人知道相同信息
- ✅ 知识积累和共享
- ✅ 配置简单
- ❌ 无隐私隔离

**适用场景：** 1-2 个机器人，同一助手的不同入口

---

### 模式 2：独立记忆（每个机器人分开）

**架构：**
```
飞书机器人A → ~/.openclaw/bots/bot-a/MEMORY.md
飞书机器人B → ~/.openclaw/bots/bot-b/MEMORY.md
飞书机器人C → ~/.openclaw/bots/bot-c/MEMORY.md
```

**特点：**
- ✅ 每个机器人独立记忆
- ✅ 隐私隔离
- ✅ 互不干扰
- ❌ 知识无法共享

**适用场景：** 3-5 个机器人，不同业务线

---

### 模式 3：混合记忆（推荐）⭐

**架构：**
```
公共记忆库（所有机器人共享）
  ├── MEMORY.md
  ├── memory/company-knowledge.md
  ├── memory/product-info.md
  └── memory/tech-docs.md

↓ 每个机器人都可以访问

机器人A私有记忆
  ├── MEMORY-A.md（对话记录）
  └── memory-A/conversations.md

机器人B私有记忆
  ├── MEMORY-B.md（对话记录）
  └── memory-B/conversations.md
```

**特点：**
- ✅ 公共知识共享
- ✅ 个性化记忆独立
- ✅ 灵活配置
- ✅ 隐私保护
- ⚠️ 配置稍复杂

**适用场景：** 5+ 机器人，大多数实际应用

---

## 配置对比

| 特性 | 统一记忆 | 独立记忆 | 混合记忆 |
|------|----------|----------|----------|
| 配置复杂度 | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 知识共享 | ✅ 完全共享 | ❌ 完全隔离 | ✅ 部分共享 |
| 隐私隔离 | ❌ 无 | ✅ 完全 | ✅ 选择性 |
| 扩展性 | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **推荐度** | ⭐⭐⭐ | ⭐⭐⭐⭐ | **⭐⭐⭐⭐⭐** |

## 快速选择指南

| 机器人数量 | 推荐模式 |
|-----------|----------|
| 1-2 个 | 统一记忆 |
| 3-5 个，不同业务 | 独立记忆 |
| 5+ 个 | 混合记忆 |
