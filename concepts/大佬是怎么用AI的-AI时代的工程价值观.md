## 二、Christoph 的 AI 使用方法

**工具：** Codex CLI + GPT-5.5 high

**工作流：**
1. **提案阶段先迭代** — "先让AI读上下文，做提案，提问题"，不直接撸代码
2. **Bug修复先写失败测试** — "red → green" 约束大幅提升修复质量
3. **代码审查用 Codiff** — `codiff -w` 让AI生成改动走读
4. **小模块直接合并** — 测试靠谱+结构OK+非关键模块 = 直接过

## 三、AI时代 6 条工程价值观

### 2.1 强大的所有权（Strong Ownership）

- 有深度产品理解的人，用AI快**10倍**
- **没背景知识的人，用同样AI只会生成更多噪音**
- → **协调成本变贵了**
- → 最有效率的团队会变得很小（2-3人），各自维护独立仓库

### 2.2 品味最重要（Taste, Taste, Taste）

> "我对 bullshit 的容忍度很低，有了 AI，每个人都能全天候批量制造 bullshit。求求你们，停下来。"

- AI时代最大问题：**不是AI写不好，而是AI太能写了**
- 一个模糊prompt，30秒能吐出功能齐全但你不需要的东西
- **品味 = 花时间想"什么值得做"，而不是直接开干**

### 2.3 严格护栏 + 快速反馈（Strict Guardrails & Fast Feedback）

- 跟AI agent合作 = 随时往代码库丢"不了解背景的新人"
- **lint规则、自动化测试、类型检查、严格CI** — 约束越强，AI越快越正确
- 决定AI是**1分钟完成还是60分钟完成**的分水岭

### 2.4 把上下文塞进仓库（Context in the Repo）

- 过去上下文分散在：Notion、脑子里、口头约定、生产环境、commit message
- **每个AI agent就像"新员工"，对这些地方访问权参差不齐**
- → 放一份 `CLAUDE.md` 或 `AGENTS.md` 在仓库里
- **代码可读性更重要了** — 代码量暴增的时代，越需要"一眼就能看懂的代码"

### 2.5 拥有自己的技术栈（Own your Stack）

> "有了AI，你没有理由再把产品核心部分外包给第三方依赖。"

- 每一层依赖 = 不完全理解的代码 + 不完全控制的设计 + 不完全承担的后果
- **每一行第三方代码，出问题都是你的问题**
- Christoph 自己建了一整套JS开源技术栈：数据层、国际化、UI库、工具配置…

### 2.6 Tech Lead Manager（管理思考）

- AI时代执行成本趋近于零，**个人贡献者杠杆巨大**
- 管理者如果不懂代码、不能提供技术领导力 → 存在价值消失
- **Tech Lead Manager 是AI时代最正确的管理角色**

## 四、核心洞见

| 旧思维 | AI时代新思维 |
|:---|:---|
| 代码要自己写 | 代码要会审查、判断、迭代 |
| 大团队多产 | 小团队+AI > 大团队 |
| 管理脱离代码 | 管理必须更懂代码 |
| 快速开发 | **先想清楚再干**（品味第一） |
| 依赖开源生态 | **拥有自己的技术栈** |
