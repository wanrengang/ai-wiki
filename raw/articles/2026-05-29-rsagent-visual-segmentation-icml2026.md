# 让大模型"边看边改"，视觉分割准确率直接上涨9% | ICML 2026

**量子位公众号** | 2026年5月27日 12:26

**论文：** arXiv:2512.24023（复旦大学、上海创智学院、上海交通大学）

## 核心问题：开放语义分割难在哪里

多模态大语言模型（MLLM）已经能够描述图像、回答问题、理解物体关系，但真实视觉系统需要的不只是文字答案。

**挑战：** 交互式标注、机器人感知、设计编辑、工业质检和科学图像分析都要求模型把语言理解落实到像素区域。模型必须在"语义理解"和"准确掩码"之间完成可靠转换。

用户输入可能是：
- "图中左侧正在被人拿起的物体" → 需要空间关系理解
- "找出湍急水流中保障个人安全的装备" → 需要场景常识和用途推理

**此前路线短板：** 不是"不能产生 mask"，而是"缺少确认与纠错过程"。模型如果只进行一次前向预测，就很难验证自己是否选对了目标。

---

## RSAgent 方案：让 MLLM 学会 Reason and Act

### 核心思路
RSAgent 的关键**不是把 MLLM 直接改造成 mask decoder，而是让它成为能够调度视觉工具的智能体**。

模型在每一轮：
1. 接收原图、文本指令和历史观察
2. 输出结构化推理与 tool call
3. 工具返回局部视图、候选掩码或 overlay
4. 模型基于反馈决定继续调用工具、调整提示，或提交最终答案

### 四步循环
- **Observation**：读取图像与历史结果
- **Thought**：用自然语言分析当前候选区域是否满足指令
- **Action**：选择工具和像素提示
- **Feedback**：接收工具输出并写入上下文

### 训练策略
- **Cold-start SFT**：解决"会不会按格式工作"的问题，让模型掌握工具调用语法和基本反思流程
- **Agentic RL**：解决"怎样做得更好"的问题，通过奖励信号优化多轮路径

**关键数据：**
- Cold-start SFT：约 5K 条高质量多轮推理轨迹
- RL 阶段：约 2K 个 RL 示例 + 8K 个 RefCOCOg 训练样本

## 实验结果

**基础模型：** Qwen2.5-VL-7B-Instruct
**分割工具：** SAM2-large

### ReasonSeg 基准
| 方法 | gIoU |
|------|------|
| RSAgent | **66.5%** |
| Seg-Zero-7B | 57.5% |
| 提升 | **+9.0%** |

### RefCOCOg 基准
| 方法 | 平均 cIoU |
|------|----------|
| RSAgent | **81.5%** |
| test split | 81.8% |

### 消融实验（ReasonSeg test）
- 无训练的 tool-agent：30.1 cIoU
- + Cold-start SFT：55.4（+25.3）
- 仅 RL：54.3
- Cold-start SFT + RL：**57.9**

**结论：** 先让模型学会规范工具调用，再通过强化学习优化长程决策，是 RSAgent 成立的关键。

### 奖励设计
- 移除 final reward：57.9 → 48.3（-9.6），说明最终掩码质量仍是核心目标
- 移除 process reward：鼓励模型在中间步骤持续改进，而不是盲目增加工具调用次数

## 核心价值

RSAgent 展示了一条从"看图问答"走向"视觉行动"的路径：

**模型可以围绕文本目标：**
1. 持续观察
2. 调用工具
3. 接受反馈
4. 修正假设
5. 把最终判断落实到图像像素

**适用场景：**
- 数据标注：减少人工反复试错
- 机器人感知：执行前重新确认目标区域
- 设计编辑和内容生产：自然语言意图转化为可编辑区域
- 科学图像分析：可回看、可复核的中间过程

## 核心创新总结

| 要素 | 描述 |
|------|------|
| 任务类型 | 开放语义文本引导分割（text-guided segmentation） |
| 核心机制 | 多轮工具调用 + 迭代修正 |
| 基础模型 | Qwen2.5-VL-7B-Instruct |
| 分割工具 | SAM2-large |
| 训练方法 | Cold-start SFT + Agentic RL |
| 关键创新 | 把分割从静态预测变成动态交互的 Agent 行为 |

**突破：** RSAgent 证明了多模态大模型可以从"结合文本与图像内容"进一步走向"在像素空间中推理、行动和自我修正"。
