## 主要章节

### 1. 准备工作

#### 1.1 Claude Code vs Codex 区别
- 两者都是编程 Agent 起家，逐步走向通用 Agent
- Anthropic 出品 Claude Code，OpenAI 出品 Codex
- Codex 后发优势：浏览器操控、Computer Use、image2 生图、手机端、Hook
- 作者日常：办公用 Codex，写作用 Claude Code + Opus 4.6

#### 1.2 基础上手
- 需要 ChatGPT 账号（免费额度少）
- 官网下载：https://chatgpt.com/zh-Hans-CN/codex/get-started/

#### 1.3 界面介绍
- 左侧：管理对话和任务列表
- 中间：聊天窗口
- 右侧：结果展示

### 2. 能力一：本地文件操作

#### 2.1 项目文件夹
- 选中本地文件夹，Codex 可读取和操作其中所有文件
- 需要授权

#### 2.2 三种权限模式
- 自动审查模式（推荐新手）
- 手动审批模式
- 全权信任模式

#### 2.3 实战案例
- 让 Codex 按视频内容重命名 80+ 视频素材
- 自动抽取关键画面判断内容，生成缩略图对照
- 遵循"序号-人物-场景-行动"命名格式

#### 2.4 多对话并行
- 同一项目下可开多个对话
- 可用 ffmpeg 拼接视频

#### 2.5 创建新项目
- 本地自动同步创建文件夹
- Agent 产生的文件（pdf/md/ppt）存在本地

### 3. 常用功能

#### 3.1 上下文管理
- 界面小圈显示上下文窗口使用量
- 快满时自动压缩
- 可手动 `/压缩`

#### 3.2 额度状态
- 设置中查看剩余额度
- 或输入 `/状态` 在对话中持续显示

#### 3.3 模型选择
- 速度：快速 = 加急通道，消耗更多额度
- 可选多种模型，一般用最新模型
- 智能程度：中度足够，可选高度
- 语音输入功能（但转录速度不如专业输入法）

### 4. 能力二：命令行工具使用

#### 4.1 装环境
- **常用依赖**：nodejs、git 等
- **新 Agent**：可以让 Codex 安装 Hermes、Claude Code 等
- **软件应用**：下载和卸载，如 Cursor、Antigravity
- **Skills/MCP/CLI**：找到链接发过去即可安装

#### 4.2 并行任务处理
- 同时运行多个任务
- 状态指示：转圈=执行中，蓝色点=完成

### 5. 能力三：持久记忆

#### 5.1 全局 agents.md
- 位置：设置 → 个性化 → 自定义指令
- 所有项目对话时第一时间作为上下文
- 可手动写或让 Codex 记住

#### 5.2 项目级 agents.md
- 只在当前项目生效
- 让 Codex 根据项目内容自动生成

#### 5.3 自动记忆（实验性）
- 对话结束/任务完成/闲置后自动总结
- 限制：太短对话不记、额度太低不记
- 不建议手动修改

### 6. 能力四：图片生成

#### 6.1 计划模式
- 先整理需求、生成完整计划，再执行
- 适合复杂网站开发任务

#### 6.2 随时纠偏
- 任务进行中可随时补充修改意见
- 不打断模型运行，下次工具调用时发送

#### 6.3 分叉 Fork
- 从任意 AI 回复处新开对话
- 适合聊岔了想回头的情况

#### 6.4 桌面宠物
- 设置 → 外观 → 可选小宠物
- 脑袋上显示 Codex 正在做什么

#### 6.5 内置预览浏览器
- 右侧栏直接打开
- 可批注具体元素

### 7. 能力五：插件

#### 7.1 一键部署上线
- Vercel / Netlify
- 无域名选 Netlify（国内访问更好）

#### 7.2 操控类插件
- **Browser Use**：新开浏览器去点击、翻页、截图、填表单
- **Chrome 插件**：操控已登录浏览器，不新开窗口，可后台执行
- **Computer Use**：最强大，操控浏览器和 App（仅 Mac）

### 8. 能力六：把流程沉淀成 Skills

#### 8.1 装别人写好的 Skill
- 用 find-skill 元 Skill 搜索社区 Skill

#### 8.2 自己创建 Skill
1. 直接告诉 Codex 想创建什么，通过讨论打磨
2. 先跑通流程，再形成 Skill（推荐）

### 9. 能力七：MCP

- 设置 → MCP 服务器
- 把 MCP 链接发给 Codex，它会引导授权

### 10. 能力八：自动化定时任务

#### 10.1 创建方式
- 方式一：自动化面板创建
- 方式二：直接告诉 Codex（推荐）

**案例**：每 3 天自动执行 GitHub 热门项目推荐 Skill，产出图文发群里 = 自动社交媒体运营机器

### 11. 手机操控 Codex

- 手机 ChatGPT + 电脑 Codex 联动
- 设置简单，可连多台电脑
- 相当于随时随地给 Codex 下发任务

## 核心观点

1. **我们很多事，真的可以交给 Agent 做了**
2. **我们已经从"问 AI"走到了"管理 AI"的阶段**

现在用 Codex 这类 Agent，每个人都得像领导一样：
- 准备上下文和工作环境
- 指明任务目标
- 检查计划、监督过程、验收结果
- 把好方法沉淀成可复用技能
- 把反复要做的事设成自动化
