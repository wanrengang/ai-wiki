# 各路大厂，集体向微信 AI 缴械

> 来源：宇宙编辑部（特工宇宙）
> 日期：2026年6月9日 20:52
> 链接：https://mp.weixin.qq.com/s/cU-z9PvhztFoMQYwsrg1jw

## 事件概述

24小时内，京东、美团、携程、同程、滴滴、途虎、肯德基等多家大厂集体宣布接入微信AI生态，成为首批内测合作伙伴。

## 接入格局

| 领域 | 首批接入方 | 落地场景 |
|------|-----------|---------|
| 电商 | 京东 | 电商、外卖、物流全链路AI Agent |
| 零售 | 多点数智+胖东来 | 对话式购物、支付、履约 |
| 本地生活 | 美团 | 外卖服务被微信AI直接调用 |
| 餐饮 | 肯德基 | 语音点餐 |
| 旅游出行 | 携程、同程 | 机票酒店行程 |
| 酒店 | 万达酒店及度假村 | 近300家酒店库存价格AI直调 |
| 出行 | 滴滴 | 快车、专车、特惠车 |
| 汽车 | 途虎养车 | 找门店 |
| 硬件 | 华为、荣耀、小米、OPPO、vivo | A2A协作，语音发微信/打视频 |

## 大厂为何"没得选"

### 微信的接入策略：低门槛 + 隐形压力

1. **自动模式**：开关一开，授权读小程序源码，零额外开发
2. **开发模式**：功能拆成Skill精细定制
3. **不接入的后果**："两种模式均关闭，微信AI将无法推荐及调用此小程序"——在14亿月活新入口里彻底消失

### 腾讯的开放逻辑

> "我们不会控制所有的入口，我们只提供底层链接" —— 马化腾

- 京东还是京东，美团还是美团
- 以独立智能体身份做A2A协作，不被管道化
- 腾讯有流量，缺供应链/运力/库存；大厂有履约能力，缺入口——互补

## 真正交出的是什么

**不是数据，不是控制权，而是「入口」——App界面**

- 过去二十年：用户习惯=壁垒（美团首页、携程下单路径）
- AI时代：用户说"帮我订周五去北京的机票"，入口变了
- UI和交互体验在Agent面前不再重要
- 让出"用户必须打开我的App"特权，换取"AI调度时代不被跳过"的资格

## 阿里 vs 腾讯的不同路线

| | 千问（阿里） | 微信AI（腾讯） |
|--|------------|--------------|
| 模式 | 超级Agent内嵌自有服务+第三方品牌，强控制 | 独立合作，A2A协作，不控制入口 |
| 优势 | 淘宝/支付宝/飞猪/高德闭环 | 14亿月活+开放生态 |
| 劣势 | 既当裁判又当运动员 | 依赖合作伙伴履约 |

## 被忽视的群体：中小开发者

- 现有小程序收入靠用户打开页面后的广告
- AI直接调用接口返回结果 → 用户不打开页面 → 广告展示归零
- 这笔账还没人能算明白

## 深层影响分析

### 1. 本地生活入口重构

**「人找服务」→「服务找人」**

以前：打开App → 搜索 → 筛选 → 下单
未来：说一句需求，AI在生态里理解、调度、执行

### 2. AI时代的流量中心转移

- 谁更懂你、办事更快、覆盖场景更全，谁就是新流量中心
- 腾讯+京东+美团+滴滴的组合短期难复制
- 阿里千问、字节豆包各守山头

### 3. 上一次微信大改版是小程序，养出了百亿级生意

### 4. 对中小开发者的警示

- 必须开发对AI友好的小程序
- 注册对AI友好的新小程序（有人已开始批量注册）

- 阿里千问、字节豆包各守山头
- 大厂站队未结束
- 中小开发者大多还在观望
