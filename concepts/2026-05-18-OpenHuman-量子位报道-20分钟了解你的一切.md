# OpenHuman用20分钟了解你的一切，存成卡帕西式知识库

> 来源：[量子位 QbitAI](https://mp.weixin.qq.com/s/YC4Xt8yKk3yxpIqJMwBZlw)  |  2026-05-16

## 核心定位

OpenHuman = 个人 AI 超级智能体。核心理念：让 AI 不再需要你教，而是主动了解你。

## 核心三步链路

**连接 → 抓取 → 记忆树**

### 第一步：连
- 支持 118+ 第三方服务
- 一键 OAuth 授权，不需要手动生成 API Key
- Notion、GitHub、Slack、Stripe、Drive 等主流工具全覆盖

### 第二步：抓
- 每 20 分钟自动轮询所有已连接账户
- 新邮件、日程变更、代码提交、文档更新全拉到本地
- 不需要写 prompt 或轮询脚本，Agent 自己知道什么时候该刷新

### 第三步：记
- 抓来的数据清洗压缩，切成 ≤3000 Token 的 Markdown 片段
- 按主题、时间线、关联对象做评分和层级摘要，折叠成记忆树
- 本地 SQLite 数据库存储
- 同一份数据同步生成 .md 文件，落成 Obsidian 兼容知识库，可直接用 Obsidian 打开浏览和编辑

## TokenJuice 令牌压缩

- 工具调用结果、网页抓取、邮件正文在送 LLM 前先过压缩
- HTML→Markdown、长 URL 缩短、非 ASCII 字符清理、冗余信息去重
- Token 消耗可砍掉 80%
- 三层规则叠加：内置默认规则 + 用户自定义规则 + 项目级规则，JSON 文件存储，改了不用重新编译

## Mascot 桌面吉祥物

- 虚拟形象可作为独立参会者加入 Google Meet 会议
- 开会时旁听记要点，离场后后台继续执行待办任务
- 潜意识循环机制：不主动交互时也会自己加载待办、读取近期记忆、自主决定还有什么可以干

## 解决的用户痛点

1. **API 密钥一大堆** → 一个账号一键搞定所有，不用反复注册
2. **平台数据分散难整合** → 内置 118+ 主流应用一键互联，自动拉取全平台数据同步
3. **上下文臃肿导致 AI 越用越卡** → TokenJuice 压缩，最高节省 80% token 消耗与响应延迟

## 评价

OpenHuman 解决了"AI 不懂我"的问题——自动拉取用户在各平台的数据，形成本地记忆树，让 AI 上岗第一天就能干活。

但注意：目前是 Early Beta 阶段，无 CLI/API，只有桌面 GUI；记忆树等特性尚待真实生产环境验证。
