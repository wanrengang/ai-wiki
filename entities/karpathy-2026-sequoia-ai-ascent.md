# Karpathy 2026 红杉 AI Ascent 演讲

## 核心观点

### 1. 编程的 AGI 拐点已到
Karpathy 自述：2025年12月密集使用最新模型时，连续几天没手动改过一行代码。「我不断要求 Claude Code 生成更多代码，效果都不错，现在我已经记不清上次自己亲自修改代码是什么时候了」。

关键心理机制：人对 AI 的信任不是线性增长，而是在某个时刻**突然跨越**的。当 AI 代码错误率降到某个临界点，行为模式从逐行检查转为完全信任。

### 2. Vibe Coding → Agentic Engineering
- **Vibe Coding**：提升所有人能力下限，门槛降低，人人都能写代码
- **Agentic Engineering**：保持专业质量标准的前提下实现效率革命
- 两者互补，面向不同人群

顶尖工程师借助 AI Agent，效率提升远不止十倍。而大部分公司还没有适应这个变化。

### 3. LLM 是第三种计算范式
Karpathy 将软件分为三层：
1. **传统代码**：if-else，严格执行，结果可预测
2. **神经网络权重**：数据样本，模型学习"软规则"
3. **LLM**：你写需求，上下文窗口是"代码编辑器"，提示词是"编程语言"

他甚至推演：未来神经网络可能成为计算的主进程，传统 CPU 退化为协处理器。

### 4. 三个正在发生的范式切换例子
1. **菜单生成 APP**：拍菜单照片 → LLM 直接返回带菜品的渲染图，无需 OCR/传统模块
2. **install.sh 替代**：把安装文档交给 AI Agent，LLM 是"高级英语解释器"
3. **知识库新方式**：对任意来源、任意格式的非结构化数据生成新知识组织方式

### 5. LLM 能力差异性的两个解释框架
- **可验证性**：数学、代码天然可验证，进步飞速；日常常识、审美难以构建验证环境，提升慢
- **经济因素**：代码领域商业价值大，实验室大量投入；商业价值不显眼的方向被忽视

金句：「你要么身处数据分布的高速公路之中，顺风顺水；要么就像拿着砍刀，在丛林里越野。」

### 6. 给创业者的建议
在垂直领域构建「验证器 + 数据」飞轮系统。谁拥有特定领域的验证环境和高质量数据，谁就能通过微调获得超越通用模型的性能。

口腔领域例子：拥有大量多样化口腔数据 → 整合 → 效果不错的结果。

### 7. Agent 原生世界愿景
- 每个人和每个组织都有智能体代表
- Agent 与 Agent 直接交互，人类负责设定目标和顶层判断
- 人类仍负责：美学设计、逻辑判断、品味把控、需求准确性

最大困难不是写代码，而是部署（DNS、配置等）——这些基础设施假设操作者是人类，对 Agent 处处是问题。
