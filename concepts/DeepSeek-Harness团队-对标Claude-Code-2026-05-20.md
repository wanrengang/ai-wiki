# DeepSeek 正在组建 Harness 团队，对标 Claude Code

> 来源：甲子光年（2026-05-20）
> 作者：王博
> 原始链接：https://mp.weixin.qq.com/s/DfsdbjgzcHJQR6rcA0ROWA

## 核心消息

知情人士透露，**DeepSeek 内部正在组建一个新的 Harness 团队**，方向是代码智能体产品，**对标 Anthropic 旗下的 Claude Code**。

DeepSeek 资深研究员**陈德里**也在社交媒体证实此事，称"简单来说就是对标 Claude Code，做 DeepSeek Code Harness"。

---

## 招聘信息

DeepSeek 开放了两个关键岗位：
- **Harness 产品经理**
- **Harness 研发工程师**
- 工作地点：北京海淀区融科资讯中心

岗位描述中最显眼的核心公式：
> **Model + Harness = Agent**

---

## 为什么强调 Harness？

### Harness 是什么？

原本指"测试线束"或"运行框架"，在 Agent 语境中，它是一套让模型真正行动起来的外部系统：
- 上下文管理
- 工具调用
- 任务规划
- 文件读写
- 代码修改
- 终端执行
- 反馈回收
- 评测闭环

**Model 负责理解、推理和生成，Harness 负责把这些能力接入真实环境。**

### 岗位核心职责

> "我们正在把 DeepSeek 的前沿模型能力，转化为领先的 Agent 产品。这其中除模型本身以外的所有工作，都属于 Harness 的范畴。"

- 规划 DeepSeek Harness 产品路线图
- 串联研究员、工程师、开源社区与终端用户
- 与模型训练团队的研究员深度沟通，实现模型与 Harness 的共同进化
- 参与"DeepSeek 桌面端 Agent 产品"全过程
- 定义 DeepSeek 对 Harness 的理解

---

## DeepSeek 为什么必须补上 Code Harness？

### 1. 代码能力强 ≠ 开发者真正用起来

真正改变开发者工作方式的不是单独的 Claude 模型，而是 Claude Code；不是单独的 GPT 模型，而是 Codex。不是聊天框里的代码回答，而是一个能够进入终端、理解项目、读写文件、运行命令、修复错误、管理 Git、调用工具的工程智能体。

### 2. DeepSeek-TUI 的启示

开发者社区已围绕 DeepSeek 模型长出了"DeepSeek 版 Claude Code"（DeepSeek-TUI 开源项目），说明：
- DeepSeek 模型在开发者心智里已具备做代码 Agent 的基础
- DeepSeek 缺的不是模型关注度，而是**官方 Harness**

### 3. 社区项目的边界

第三方开源项目难以：
- 掌握模型内部能力演化节奏
- 反向决定模型如何训练
- 把海量真实任务反馈系统性注入模型改进

### 4. 官方 Harness 的优势

- 模型团队协作
- 接口设计权
- 训练数据闭环
- 内部真实任务场景
- 对开发者生态的长期运营能力
