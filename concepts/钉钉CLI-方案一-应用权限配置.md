# 钉钉CLI方案一：应用Token实现全员数据访问

## 背景

当前架构：OpenClaw服务器上只部署了一份钉钉CLI（dws），使用单一Token（老万的账号）。

问题：其他组织成员操作钉钉时，都是用老万的Token，查到的都是老万的数据，无法实现"各自用各自Token查各自数据"。

## 方案一：应用Token（代理访问）

### 核心思路

用企业的**应用身份**而不是用户身份去访问数据。

- 管理员在钉钉开放平台给应用申请"通讯录只读"等权限
- 权限范围配置为"全体员工"
- 应用用自己的 client_id + client_secret 获取 token
- 这个 token 可以查询企业内任何人的信息

### 实现步骤

#### 第一步：确认应用权限范围

在钉钉管理后台完成：

1. 进入 **应用管理** → 找到 `DING_DWS_CLAW`
2. 点击 **权限管理**
3. 确认已开通以下权限：
   - ✅ 通讯录只读
   - ✅ 日历只读（如果需要查日程）
   - ✅ 考勤只读（如果需要查打卡）
   - ✅ 其他需要的权限...
4. **权限范围**需设置为 **"全体员工"**（而不是"部分员工"）

#### 第二步：切换到应用Token

权限配置好后，用应用凭证登录：

```bash
# 管理员需要在服务器上执行一次
dws auth login --client-id <APP_KEY> --client-secret <APP_SECRET>
```

这会用**应用身份**获取Token，而不是用户身份。

#### 第三步：验证

登录后查询测试：

```bash
# 查一个其他同事的信息（比如 userId: 383806）
dws api GET /v1.0/contact/users/383806
```

如果能查到，说明应用权限生效 ✅

**当前的应用是谁创建的？**

- 如果是你们自己创建的 → 你们就是管理员，可以直接去后台配置权限
- 如果是第三方ISV提供的 → 需要联系他们开通权限
