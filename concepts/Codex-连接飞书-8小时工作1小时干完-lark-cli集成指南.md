## lark-cli 是什么

lark-cli 是 Codex 和飞书之间的一条操作通道。

平时 Codex 只能根据你贴给它的内容做总结、改写、分析。装上 lark-cli 后，它就有机会直接读飞书里的文档、会议纪要、表格、日程和任务。

能读什么、能写什么，取决于你给它的权限。

## 安装步骤

**安装 lark-cli：**
```powershell
npm install -g @larksuite/cli
```

**安装 Agent Skill（建议）：**
```powershell
lark-cli agent install
```

**初始化配置：**
```powershell
lark-cli init
```

**登录授权：**
```powershell
lark-cli auth login --recommend
```

**验证安装：**
```powershell
lark-cli auth status
```
看到 user 或 bot 身份 ready，基本就通了。

## 4 个高价值场景

### 场景 1：开完会，行动项自动落地

开完会最烦的是后半段：翻纪要、摘待办、确认负责人、补截止日期、建任务。

**操作链路：**
```
会议纪要 → 提取行动项 → 识别负责人和截止时间 → 生成预览 → 创建飞书任务 → 写回行动项追踪表
```

**参考 prompt：**
> 用 lark-cli 读取最近一次项目会议纪要，提取所有行动项。请整理成：事项、负责人、截止时间、优先级、来源原文。先给我预览，确认后创建飞书任务，并在会议纪要下方追加"行动项追踪"表格。

### 场景 2：文档批注自动执行，并回复已完成

批注经常比正文还烦——"这里改正式一点""补一个截止日期"——没什么判断难度，但很耗手。

可以让 Codex 把批注分类，明确能改的直接处理，改完回复"已完成"。

### 场景 3：内容创作者收集信息，整理成选题和长文大纲

做内容的人不缺素材，缺的是把素材捞出来。很多观点散在群聊、会议纪要、客户反馈、表格和文档里。

接上飞书后，可以让 Codex 读取这些素材，整理成选题和大纲。

### 场景 4：周报、日报和项目进展自动生成

周报难写吗？其实不难。烦的是找材料：本周做了什么、参加了哪些会、任务推进到哪里。

可以让 Codex 先把材料捞出来，再生成周报。

## 安全使用建议

涉及写入、发送、审批的操作，先 dry-run 或先给预览，不要直接执行。

lark-cli auth scopes     # 查看权限
lark-cli calendar list   # 查看今天日程
