# 钉钉CLI多用户Token实现方案

## 背景问题

当前架构：
- OpenClaw 服务器只部署一份
- 钉钉 CLI (dws) 只部署一份，只有一个 Token（老万的账号）
- 其他组织成员无法使用各自的Token查各自的数据

## 方案一：应用Token（代理访问）— 推荐

### 核心思路

用企业的**应用身份**而不是用户身份去访问数据。应用用自己的 client_id + client_secret 获取 token，可以查询企业内任何人的信息。

### 实现步骤

#### 第一步：确认应用权限范围

在钉钉管理后台（管理员需完成）：

1. 进入 **应用管理** → 找到 `DING_DWS_CLAW`
2. 点击 **权限管理**
3. 确认已开通以下权限：
   - ✅ 通讯录只读
   - ✅ 日历只读（如果需要查日程）
   - ✅ 考勤只读（如果需要查打卡）
   - ✅ 其他需要的权限...
4. **权限范围**需设置为 **"全体员工"**（而不是"部分员工"）

#### 第二步：切换到应用Token

权限配置好后，用应用凭证登录：

```bash
# 管理员需要在服务器上执行一次
dws auth login --client-id <APP_KEY> --client-secret <APP_SECRET>
```

#### 第三步：验证

```bash
# 查一个其他同事的信息
dws api GET /v1.0/contact/users/383806
```

## 方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| 方案一：应用Token | 管理员帮任何人查数据 | 简单，无需用户配合 | 不是真正的用户级隔离 |
| 方案二：多用户Token | 每个人只能查自己的数据 | 完全隔离 | 需要改造CLI |
