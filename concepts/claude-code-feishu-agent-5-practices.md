作者分享了5个实际应用案例，展示了Agent与飞书CLI结合的强大办公能力。

## 5个核心用法

### 1. 给每个会议系列养一个跨场次的知识库

**痛点**：培训会、周会、复盘会等重复性会议结束后，妙记仅躺在飞书里，无人追踪。选题会长线选题讨论后假期回来就无人再提；新员工也无法快速了解历史会议沉淀。

**方案**：让Agent调用飞书CLI直接拉取历史会议数据，自动整理成结构化文档，包含：
- 会议清单
- 选题转化分析
- 会议节奏
- 决策机制
- 现象与建议

**价值**：重复会议的经验留存可复用到各个方面，积累成固定知识库，新员工入职培训也可用历史数据快速入门。

---

### 2. Agent+飞书做工作整体复盘

**痛点**：工作数据分散在私信、群聊、会议、妙记、日程、任务、邮件、文档、OKR等多个平台，个人无法对自己做横切面分析。

**方案**：Agent分8路并行抓取过去7天/30天/90天的全量数据，生成全面工作复盘文档：
- 一句话总结
- 时间投入分布
- 协作情况
- 主要项目与产出
- Agent洞察与建议

**价值**：可发现排期遗漏、文件传输助手待整理等问题，生成的文档可直接用于季度汇报，替代凑字数的水货报告。

---

### 3. 重复性对接流程自动化

**案例：博主对账自动化**

**痛点**：每月对接几百个博主，经纪人仅3人，需在群里反复沟通对账明细，重复沟通极多。

**方案**：开发飞书机器人，自动化流程如下：
1. 把博主和机器人拉进同一群聊
2. @机器人发起对账
3. 机器人查询多维表格中的项目数据
4. 在群内@博主展示所有合作项目、金额、备注
5. 博主反馈后，机器人自动分流：
   - 金额不对 → 通知商务核对刊例
   - 漏单 → 通知财务补单
   - 没问题 → @负责人确认完成

**价值**：全程自动化，只需在群里@机器人，零人工干预。可复制到客户回款核对、订单异常处理、合同到期提醒、年终数据核对等场景。

---

### 4. 生成可协同的画板

**新能力**：飞书最近推出的画板功能，Claude Code可直接生成飞书原生画板。

**对比传统方式**：生成HTML网页或图片需单独分享、无法协同编辑。

**方案**：让Claude Code调用飞书CLI，直接生成飞书画板，分享给团队后任意人可拖动、改文字、加节点、改连线。

**场景**：
- 开会时当场讨论架构/流程图，丢到群里边改边讨论
- 将文字版流程（如AIFUT大会观众入场流程）一键转为流程图
- 画板可替代PPT，无需进入PPT软件即可协同

**核心价值**：协同、协同、协同。

---

### 5. 自动报销和审批

**痛点**：报销流程繁琐，从发票整理到审批完成涉及多个系统。

**方案**：让Claude Code自动完成全流程：
1. 去指定邮箱搜索发票
2. 整理发票信息
3. 查询知识库中的报销SOP
4. 照SOP在飞书上发起报销申请（发起前与用户确认）
5. 审批人端也可由Agent自动核对信息、给审批意见、完成审批通过

**价值**：整个报销流程从发票整理到审批完成，全程与Claude Code对话，完全不需要打开飞书手动操作。财务惊了。
