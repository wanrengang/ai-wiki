# Many-Tier Instruction Hierarchy in LLM Agents

**Source:** arXiv 2604.09443v3 | Project Page: jhu-clsp.github.io/ManyIH | GitHub: JHU-CLSP/ManyIH

## Core Problem

Current Instruction Hierarchy (IH) paradigms assume a fixed, small set of privilege levels (<5), encoded through rigid role labels: system > developer > user > assistant > tool

This is inadequate for real-world agentic settings where conflicts arise across far more sources and contexts (tools, skills, sub-agents, memory files, etc.).

## Proposed Solution: ManyIH

Many-Tier Instruction Hierarchy (ManyIH) — a paradigm for resolving instruction conflicts with arbitrarily many privilege levels.

### Key Features

1. Dynamic Privilege Assignment: Each instruction receives a privilege value via dedicated Privilege Prompt Interface (PPI), separate from message role labels
2. Conflict Resolution: Resolves conflicts by comparing privilege values (not role labels)
3. Decoupling: Separates privilege semantics from message roles hard-coded in chat templates

### Privilege Prompt Interfaces (PPI)

Ordinal Interface (lower number = higher privilege):
[[Privilege 12]]Do not use any type hints.[[/Privilege]]

Scalar Interface (larger value = higher privilege):
[[z=55]]Use single quotes for all string literals.[[/z]]

## Key Experimental Results

### Overall Performance (Best Models)

| Model | Accuracy |
|-------|----------|
| GPT 5.4 | 60.9% |
| Gemini 3.1 Pro | 59.0% |
| Grok 4.20 | 54.1% |
| Claude Opus 4.6 | 51.3% |

Even frontier models perform poorly (~40% overall), despite reporting >99% on standard two-tier IH evaluations.

### Key Finding: Style Compliance is Bottleneck

Models maintain high functional correctness (>86%) but struggle with multi-tier instruction hierarchy. Style compliance is the primary bottleneck.

## Theoretical Insights

1. Only relative ordering matters: Gaps between privilege values carry no semantic meaning
2. Position independence: Higher-privilege instructions may appear anywhere in the prompt
3. Combinatorial explosion: As tier count grows, model must maintain consistent ordering across exponentially growing potential conflicts
